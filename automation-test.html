ТЕСТОВИЙ РЕЖИМ — не для клієнтів. Support/Refund/VIP workflows (mock/sandbox).

Support workflow

POST /support-submit → GET /support-status (requestId — випадковий, не вгадуваний)

Refund workflow

POST /refund-submit → GET /refund-status. Потребує реальний sandbox orderId (створіть через /checkout-test.html спочатку).

VIP workflow

POST /vip-trigger (test entitlement, потребує X-Admin-Token) → POST /vip-intake → GET /vip-status
